# Скилл: Query (Отбор примеров)

Ты — стратег отбора данных. Выбираешь из unlabeled pool те примеры,
которые принесут максимум пользы для модели.

## Вход

- `pool_df` — неразмеченные данные (unlabeled pool)
- `strategy` — стратегия отбора: `entropy`, `margin` или `random`
- `n_samples` — сколько примеров отобрать (по умолчанию 10)
- Обученная модель из fit (self.pipeline_)

## Стратегии

### 1. Entropy Sampling (`entropy`)

Выбирает примеры с максимальной энтропией предсказанных вероятностей.
Высокая энтропия = модель максимально не уверена.

```
H(x) = -Σ p(y|x) * log(p(y|x))
```

- Для бинарной задачи: максимум энтропии при p=0.5
- Для мультикласса: максимум при равномерном распределении
- Выбираем top-n по убыванию энтропии

Когда хорош: когда нужно найти границы решений, хорошо работает
для мультикласса.

### 2. Margin Sampling (`margin`)

Выбирает примеры с минимальным margin — разницей между
вероятностями двух самых вероятных классов.

```
margin(x) = P(y1|x) - P(y2|x)
```

где y1 — самый вероятный класс, y2 — второй по вероятности.

- Маленький margin = модель колеблется между двумя классами
- Выбираем top-n по возрастанию margin (самые маленькие)

Когда хорош: когда есть пары классов которые путаются,
для бинарной классификации практически идентичен entropy.

### 3. Random Sampling (`random`)

Выбирает случайные примеры из pool. **Это baseline** для сравнения.

- Используй фиксированный random_state для воспроизводимости
- Должен быть честный random — без какой-либо сортировки

Зачем нужен: показать что AL стратегии реально лучше случайного отбора.

## Что делать

### Шаг 1: Получи вероятности

- Примени self.pipeline_ к pool_df
- Получи predict_proba → массив [n_samples, n_classes]
- Проверь что вероятности нормальные (не NaN, не все одинаковые)

### Шаг 2: Посчитай скоры

В зависимости от стратегии посчитай informativeness score
для каждого примера в pool.

### Шаг 3: Отбери top-n

- Отсортируй по score
- Возьми top-n индексов
- Верни именно **индексы из pool_df** (не позиции в массиве)

### Шаг 4: Диагностика

Покажи:
- Распределение scores по pool (гистограмма)
- Score отобранных vs остальных
- Примеры отобранных записей (первые 5)

## Функция в al_agent.py

`query(self, pool_df, strategy='entropy', n_samples=10) -> np.array`

Возвращает массив индексов из pool_df.

Внутри:
- if strategy == 'entropy': считай энтропию
- if strategy == 'margin': считай margin
- if strategy == 'random': random choice
- else: raise ValueError

## Правила

- Модель ДОЛЖНА быть обучена перед query — проверяй hasattr(self, 'pipeline_')
- random_state ВЕЗДЕ одинаковый для воспроизводимости
- Если pool меньше n_samples — верни весь pool, не падай
- Индексы должны быть из ОРИГИНАЛЬНОГО pool_df.index, не из reset_index
- Не мутируй pool_df — работай с копиями
- Логируй стратегию, n_samples, score stats для каждого вызова

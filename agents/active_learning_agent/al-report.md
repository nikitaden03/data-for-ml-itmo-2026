# Скилл: Report (Отчёт и Learning Curves)

Ты — аналитик Active Learning эксперимента. Строишь графики,
сравниваешь стратегии и доказываешь что AL экономит данные.

## Вход

- `history` — история метрик из evaluate (для каждой стратегии отдельная)
- Минимум 2 стратегии для сравнения: entropy и random
- (опционально) margin тоже

## Что строить

### 1. Learning Curves (главный график)

**Один график, несколько линий:**
- X: количество размеченных примеров (n_labeled)
- Y: метрика (accuracy или F1 macro)
- Линии: по одной на стратегию (entropy=синий, margin=зелёный, random=серый пунктир)
- Маркеры на каждой итерации
- Заголовок, подписи осей, легенда
- Опционально: доверительные интервалы если запускали несколько раз

Этот график — **самый важный артефакт** всего задания.

### 2. Сравнительная таблица по итерациям

```
Iteration | n_labeled | Entropy Acc | Entropy F1 | Random Acc | Random F1 |
    0     |    50     |    0.62     |    0.58    |    0.62    |   0.58   |
    1     |    60     |    0.68     |    0.65    |    0.64    |   0.60   |
    2     |    70     |    0.74     |    0.71    |    0.66    |   0.63   |
    3     |    80     |    0.78     |    0.76    |    0.68    |   0.65   |
    4     |    90     |    0.82     |    0.80    |    0.70    |   0.67   |
    5     |   100     |    0.85     |    0.83    |    0.73    |   0.70   |
```

### 3. Расчёт экономии (Sample Efficiency)

Это ключевой вывод. Посчитай:

**Вариант A: При одинаковом качестве — сколько примеров сэкономлено**

- Возьми финальную accuracy/F1 random стратегии (например 0.73)
- Найди на какой итерации entropy достигла того же уровня
- Разница в n_labeled = экономия

```
Random достиг accuracy=0.73 на 100 примерах
Entropy достиг accuracy=0.73 уже на 70 примерах
→ Экономия: 30 примеров (30%)
```

**Вариант B: При одинаковом количестве — насколько лучше качество**

- При n_labeled=100: entropy accuracy=0.85, random accuracy=0.73
- Разница: +0.12 (+16.4% relative)

Покажи ОБА варианта.

### 4. График: Entropy/Margin scores по итерациям

- Как менялось среднее informativeness score pool-а от итерации к итерации
- Если score растёт — модель становится увереннее (хорошо)
- Если score падает — модель всё ещё не уверена (нужно больше данных)

### 5. Финальная confusion matrix

Для лучшей стратегии — confusion matrix на последней итерации.
Визуализация через seaborn heatmap.

## Что создать

### `notebooks/al_experiment.ipynb`

Структура ноутбука:

1. **Markdown: Введение**
   - Что такое Active Learning
   - Какие стратегии сравниваем и почему
   - Описание данных и задачи

2. **Загрузка данных и начальная разметка**
   - Загрузить данные
   - Выделить test set (фиксированный)
   - Выделить initial labeled (N=50, стратифицированный)
   - Остальное → unlabeled pool

3. **AL цикл: Entropy**
   - Запуск полного цикла: 5 итераций по 10 примеров
   - Логирование метрик на каждой итерации

4. **AL цикл: Random**
   - Тот же формат, те же начальные условия
   - Тот же test set

5. **AL цикл: Margin** (бонус)
   - Опционально, третья стратегия

6. **Learning Curves**
   - Главный график: все стратегии на одном
   - Сравнительная таблица

7. **Расчёт экономии**
   - Вариант A и B
   - Визуализация

8. **Confusion Matrix финальной модели**

9. **Markdown: Выводы**
   - Какая стратегия лучше
   - Сколько примеров сэкономлено
   - Рекомендации для продакшна

### `agents/al_agent.py`

Финальный файл с полным классом `ActiveLearningAgent`.

### `data/reports/al_results.json`

## Правила

- Learning curves ВСЕГДА на одном графике для сравнения
- Random — обязательный baseline, без него нельзя
- Начальные условия ОДИНАКОВЫЕ: один и тот же initial labeled set,
  один и тот же test set, один и тот же random_state
- Экономию считай ОБОИМИ способами (при равном качестве / при равном количестве)
- Все эксперименты воспроизводимые — random_state фиксирован
- Графики: заголовки, подписи осей, легенда, сетка
- Цвета: entropy=синий, margin=зелёный, random=серый пунктир
- Если margin ≈ entropy — отметь что для данной задачи разницы нет

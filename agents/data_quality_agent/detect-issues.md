# Скилл: Detect Issues (Детектив)

Ты — детектив по качеству данных. Найди все проблемы в датасете,
количественно оцени каждую и визуализируй.

## Вход

Пользователь даёт:
- Путь к CSV-файлу
- (опционально) Название целевой переменной для анализа дисбаланса
- (опционально) Контекст ML-задачи

Если целевая переменная не указана — попробуй определить сам
(последний столбец, столбец с именем target/label/class, или спроси).

## Что искать

### 1. Пропущенные значения (Missing Values)
- Количество и процент пропусков по каждому столбцу
- Паттерны пропусков: случайные (MCAR) vs систематические (MNAR)
- Столбцы где пропусков > 50% — кандидаты на удаление
- Визуализация: heatmap пропусков (seaborn или missingno если установлен),
  bar chart процента пропусков по столбцам
- Severity: >30% = high, 10-30% = medium, <10% = low

### 2. Дубликаты (Duplicates)
- Полные дубликаты строк
- Частичные дубликаты по ключевым столбцам (если можно определить)
- Визуализация: сколько дубликатов, примеры дублирующихся строк
- Severity: >5% = high, 1-5% = medium, <1% = low

### 3. Выбросы (Outliers)
- Проверь ОБА метода: IQR и Z-score (|z| > 3)
- Для каждого числового столбца покажи количество выбросов обоими методами
- Визуализация: boxplot для каждого числового столбца,
  scatter plot если выбросов много
- Severity: зависит от процента выбросов и контекста

### 4. Дисбаланс классов (Class Imbalance)
- Только если есть целевая переменная и это задача классификации
- Imbalance ratio = count(majority) / count(minority)
- Визуализация: bar chart распределения классов
- Severity: ratio > 10 = high, 3-10 = medium, < 3 = low

## Что создать

### Файл 1: `scripts/quality_utils.py`
Создай dataclass-ы для переиспользования между скиллами:
- `IssueDetail` — одна проблема (type, severity, column, count, percentage, description)
- `QualityReport` — полный отчёт (все метрики + список issues + quality_score 0-100)
- Функция `detect_all(df, target_col=None) -> QualityReport`

Quality score считай так: начинаешь со 100, вычитаешь штрафы
за каждую проблему пропорционально severity.

### Файл 2: `notebooks/01_detect_issues.ipynb`
Jupyter-ноутбук со структурой:
1. Markdown: заголовок, описание датасета
2. Загрузка данных, базовый обзор (shape, dtypes, head)
3. Раздел "Пропуски" — анализ + визуализации
4. Раздел "Дубликаты" — анализ + примеры
5. Раздел "Выбросы" — IQR и z-score + boxplots
6. Раздел "Дисбаланс" — распределение целевой + ratio
7. Markdown: сводная таблица всех проблем с severity
8. Quality score и общий вердикт

Используй nbformat для создания ноутбука программно,
затем выполни через jupyter nbconvert --execute.

### Файл 3: `data/reports/quality_report.json`
Сохрани QualityReport в JSON для использования в следующих скиллах.

## Правила
- Не чисти данные — только находи и документируй
- Все визуализации должны быть читаемые: заголовки, подписи, размер
- Код должен работать на ЛЮБОМ CSV, не хардкодь имена столбцов
- Если датасет большой (>100k строк) — используй сэмплирование для визуализаций
- matplotlib + seaborn, без plotly (тяжёлый для ноутбука)
- Запускай код самостоятельно
- Ячейки в ноутбуке должны отработать

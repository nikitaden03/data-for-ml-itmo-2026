# Скилл: Auto Label

Ты — автоматический разметчик данных. Берёшь неразмеченный датасет
и навешиваешь метки с помощью предобученных моделей.

## Вход

Пользователь даёт:
- Путь к данным (CSV с текстами, папка с аудио, папка с изображениями)
- Модальность: `text`, `audio` или `image` (определи сам если не указано)
- Задачу разметки: классификация, NER, sentiment, object detection и т.д.
- (опционально) Список классов/меток

Если классы не указаны — определи сам на основе данных и задачи,
но **покажи пользователю и спроси подтверждение** перед разметкой.

## Определение модальности

Если пользователь не указал — определи по данным:
- CSV с текстовыми столбцами → text
- Папка с .wav/.mp3/.flac → audio
- Папка с .jpg/.png/.webp → image
- CSV с путями к файлам → посмотри расширения

## Стратегии разметки по модальности

### TEXT (основная модальность)

Поддержи минимум 2 подхода:

**Подход 1: Zero-shot classification (transformers)**
- Модель: `facebook/bart-large-mnli` или `MoritzLaurer/deberta-v3-large-zeroshot-v2.0`
- Когда: классификация текста по произвольным классам
- Возвращает: метку + confidence score

**Подход 2: spaCy NER**
- Модель: `en_core_web_sm` / `en_core_web_trf` / `ru_core_news_sm`
- Когда: извлечение сущностей (имена, организации, даты)
- Возвращает: сущности с типами и позициями

**Подход 3: Sentiment (если задача — тональность)**
- Модель: `cardiffnlp/twitter-roberta-base-sentiment-latest` или TextBlob
- Возвращает: positive/negative/neutral + score

**Подход 4: Keyword/regex-based (простой baseline)**
- Набор правил и ключевых слов
- Быстро, интерпретируемо, хорош как baseline для сравнения

### AUDIO

**Подход: Whisper → текст → text pipeline**
- openai/whisper для транскрибации
- Дальше применяй text-подходы к транскрипции
- Сохраняй и транскрипцию и метку

### IMAGE

**Подход: YOLO / CLIP**
- YOLO (ultralytics): object detection, возвращает bbox + class + confidence
- CLIP (openai/clip): zero-shot image classification по текстовым описаниям классов

## Что делать

### Шаг 1: Анализ данных

Посмотри на данные, определи:
- Модальность
- Количество записей
- Язык (для текста)
- Подходящую задачу разметки

Покажи пользователю:
```
📊 Данные:
- Файл: {path}
- Записей: {count}
- Модальность: {text/audio/image}
- Язык: {lang}

Предлагаю задачу: {classification/NER/sentiment/detection}
Классы: [{class1}, {class2}, ...]
Модель: {model_name}

Начинаю разметку?
```

**⏸️ ЖЖДИ ПОДТВЕРЖДЕНИЯ**

### Шаг 2: Установка зависимостей

Устанавливай только то что нужно для выбранной модальности.
Предупреди пользователя если модели тяжёлые (трансформеры ~1GB).

### Шаг 3: Разметка

- Обрабатывай батчами, показывай прогресс
- Для каждой записи сохраняй:
  - `label` — основная метка
  - `confidence` — уверенность модели (0.0–1.0)
  - `method` — каким подходом размечено
  - `raw_output` — полный вывод модели (для отладки)
- Если есть GPU — используй, если нет — предупреди о времени
- Если датасет большой (>10k) — сначала размеь 100 записей,
  покажи пользователю, спроси "качество ок, продолжать?"

### Шаг 4: Результат

Сохрани:
- `data/labeled/{name}_labeled.csv` — датасет с метками
- Добавленные столбцы: `label`, `confidence`, `method`, `labeled_at` (timestamp)
- Не удаляй и не изменяй оригинальные столбцы

## Что создать

- `src/labeling_utils.py` — функции авто-разметки, переиспользуемые между скиллами:
  - dataclass `LabelResult(label, confidence, method, raw_output)`
  - функция `auto_label(df, modality, task, classes, method) -> DataFrame`
- `data/labeled/{name}_labeled.csv`
- `notebooks/01_auto_label.ipynb` — процесс разметки с примерами

7. Markdown: наблюдения — где модель уверена, где нет

## Правила

- **Всегда сохраняй confidence** — без него нельзя оценить качество
- Если confidence < 0.5 — помечай метку как `uncertain`
- Не хардкодь классы — бери из параметров
- Если модель не скачивается / не работает — предложи fallback
  (zero-shot не работает → попробуй keyword-based)
- Для текста на русском: используй multilingual модели
  (`joeddav/xlm-roberta-large-xnli` для zero-shot)
- Для батчей > 1000 записей добавь tqdm-подобный прогресс через print
- Запускай код самостоятельно

# Скилл: Check Quality

Ты — контролёр качества разметки. Оцениваешь насколько хороша
автоматическая разметка и где нужна ручная проверка.

## Вход

- Размеченный датасет: `data/labeled/{name}_labeled.csv`
- (опционально) Второй набор меток для inter-annotator agreement
  (ручная разметка, второй прогон авто-разметки другой моделью)
- (опционально) Ground truth / gold standard
- Спецификация: `docs/annotation_spec.md`

## Какие метрики считать

### 1. Распределение меток (Label Distribution)

- Частота каждого класса (absolute + %)
- Энтропия распределения (насколько равномерно)
- Визуализация: bar chart + pie chart
- Сравнение с ожидаемым распределением (если известно)
- Выявление: есть ли класс < 5%? → предупреди о дисбалансе

### 2. Анализ Confidence

- Среднее, медиана, std confidence по каждому классу
- Процент записей с confidence < 0.5 (uncertain)
- Процент записей с confidence > 0.9 (уверенные)
- Визуализация: histogram confidence общий + по классам
- Визуализация: boxplot confidence по классам
- Записи с самым низким confidence — кандидаты на ручную проверку

### 3. Agreement Metrics

**Если есть два набора меток (авто + ручная, или две модели):**

- **Cohen's Kappa (κ)** — основная метрика
  - κ > 0.8 = отличное согласие
  - 0.6–0.8 = хорошее
  - 0.4–0.6 = умеренное
  - < 0.4 = плохое
- **Percent Agreement** — простой процент совпадений
- **Confusion Matrix** между двумя наборами меток
- По-классово: для каких классов agreement хуже всего

**Если есть только один набор меток (только авто):**

- Используй confidence как proxy метрику
- Рассчитай: % записей где модель "согласна сама с собой"
  (confidence > threshold)
- Если есть возможность — прогони вторую модель и посчитай
  agreement между моделями (это cross-model agreement)
- Предложи пользователю разметить 50-100 записей вручную
  для расчёта κ

### 4. Анализ ошибок (Error Analysis)

- Из записей с низким confidence — покажи 10-15 примеров
- Сгруппируй типичные ошибки:
  - Путает класс A с классом B (confusion pair)
  - Короткие тексты → низкий confidence
  - Определённые ключевые слова → неверная метка
- Предложи правила для исправления систематических ошибок

## Правила

- Cohen's κ считай через `sklearn.metrics.cohen_kappa_score`
- Confusion matrix — через `sklearn.metrics.confusion_matrix`
  + визуализация через seaborn heatmap
- Если κ < 0.6 — явно скажи что качество недостаточное для ML
  и предложи варианты улучшения
- Всегда предлагай сэмпл для ручной проверки даже если κ высокий
- Не используй accuracy как метрику agreement — она не учитывает
  случайное совпадение, для этого нужен именно κ
- Запускай код самостоятельно

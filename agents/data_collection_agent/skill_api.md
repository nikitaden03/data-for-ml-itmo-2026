# Скилл: API Fetch

## Когда использовать
Пользователь хочет свежие данные из интернета, нужно что-то, 
чего нет готовым датасетом, или данные в реальном времени.

## Как работать

### Шаг 1: Выбор API или хоста для скраипнга
Предложи 1-2 API. Для каждого:
- Название и URL
- Нужен ли ключ
- Какие данные получим
- Ограничения (rate limits, количество записей)

### Шаг 3: Обработка проблем

Типичные проблемы и решения:
- **Rate limit** → добавь time.sleep() и retry логику
- **Пагинация** → определи тип (offset, cursor, page) и обработай
- **Вложенный JSON** → используй pd.json_normalize()
- **API требует ключ** → объясни где получить, добавь переменную
- **Данных мало** → предложи альтернативный API или комбинацию
- **API не отвечает** → предложи fallback источник

### Важно
- Всегда добавляй `time.sleep()` между запросами
- Всегда обрабатывай HTTP ошибки
- Предупреждай о rate limits ДО запуска
- Если нужен ключ — давай чёткую инструкцию где получить
- Оценивай сколько запросов потребуется
- Запускай код самостоятельно
- Вначале получи 1-2 сэмпла, спроси у пользователя, подходит ли формат данных

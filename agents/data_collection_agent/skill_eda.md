# Скилл: EDA (Exploratory Data Analysis)

## Когда использовать
Пользователь хочет исследовать данные, нужны визуализации, статистика.

## Как работать

### Шаг 1: Понять что анализируем
Спроси (если неясно):
- Какой файл анализировать?
- Что интересует: общий обзор или конкретные вопросы?
- Есть ли целевая переменная?

### Шаг 2: Создание ноутбука
Генерируй **полный Jupyter-ноутбук** как артефакт.

Ноутбук создавай в формате Python-скрипта с маркерами
(пользователь сохранит как .py и конвертирует, или скопирует ячейки).

### Стандартная структура EDA-ноутбука:

```python
# %% [markdown]
# # EDA: {Название анализа}
# 
# **Датасет:** {filename}  
# **Цель:** {что исследуем}
# **Дата:** {дата}

# %%
# Установка и импорты

# Настройка графиков

# %%
# Загрузка данных

# %% [markdown]
# ## 1. Общий обзор

# %%
# Информация о типах и пропусках

# %%
# Описательная статистика
# %% [markdown]
# ## 2. Пропущенные значения

# %%
# Визуализация пропусков

# %% [markdown]
# ## 3. Распределения числовых переменных

# %%

# %% [markdown]
# ## 4. Корреляции

# %%

# %% [markdown]
# ## 5. Категориальные переменные

# %%

# %% [markdown]
# ## 6. Выбросы

# %%

# %% [markdown]
# ## 7. Ключевые инсайты
# 
# *{Claude заполняет после анализа результатов}*

# %%
# Summary
```

### Формат вывода
Создавай ноутбук как артефакт с типом `application/vnd.ant.code` и языком `python`.

Объясни пользователю:
```
Сохрани как `eda.py` и конвертируй:
  jupytext --to notebook eda.py

Или сохрани как `eda.ipynb` и открой в Jupyter.

Или просто копируй ячейки по одной в Jupyter.
```

### Адаптация под задачу
Стандартный шаблон — это база. Адаптируй под запрос:
- Временные ряды → добавь линейные графики по времени, тренды, сезонность
- Геоданные → добавь карты (plotly.express.scatter_geo)
- Текст → длина текстов, частые слова, wordcloud
- Целевая переменная → отдельный раздел с target analysis
- Сравнение групп → violin plots, grouped bar charts

### Важно
- Графики должны быть ЧИТАЕМЫЕ: заголовки, подписи осей, размер
- Не больше 15-20 графиков — иначе теряется фокус
- После каждого графика — текстовый вывод/интерпретация
- Код должен работать на ЛЮБОМ датасете (защита от пустых столбцов, 
  деления на ноль, слишком большого количества категорий)
- В конце выведи возможные пробелемы в данных и их критичность, которые следуют поправить на этапе препроцессинга
- Все Юпитер ноутбуки должны быть с отработанными ячейками и выводами - ОБЯЗАТЕЛЬНО

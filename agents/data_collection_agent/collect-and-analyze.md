# Пайплайн: Сбор и Анализ Данных

Это оркестрирующий скилл. Выполняй шаги последовательно
с паузами для подтверждения пользователем.

## Входные данные
Пользователь описывает тему/задачу. Из его запроса определи:
- Какие данные нужны
- Откуда брать (Kaggle, API, или оба)
- Как объединять
- Что анализировать

## Шаги

### Шаг 0: Подготовка
Создай виртуальную среду в проекте, устанавливай зависимости в нее

### Шаг 1: Планирование
Составь план и покажи пользователю:
```
📋 План:
1. Kaggle: скачаю {что}
2. API: получу {что} из {откуда}
3. Merge: объединю по {ключ}
4. EDA: исследую {что}

Начинаем?
```
**⏸️ ПОДОЖДИ ПОДТВЕРЖДЕНИЯ**

### Шаг 2: Kaggle Download (skill_kaggle.md)
Используй скилл kaggle-download.
После скачивания покажи результат и спроси:
```
✅ Скачал: {file}, {rows} строк, {cols} столбцов
Столбцы: [...]
Продолжаем с API?
```
**⏸️ ПОДОЖДИ ПОДТВЕРЖДЕНИЯ**

### Шаг 3: API Fetch (skill_api.md)
Используй скилл api-fetch.
После получения покажи результат:
```
✅ Получил: {file}, {rows} строк
Столбцы: [...]
Объединяю датасеты?
```
**⏸️ ПОДОЖДИ ПОДТВЕРЖДЕНИЯ**

### Шаг 4: Merge (skill_merge.md)
Используй скилл merge-datasets.
Покажи стратегию мерджа и результат:
```
✅ Объединил: {rows} строк, {cols} столбцов
Потерянных строк: {n}
Запускаю EDA?
```
**⏸️ ПОДОЖДИ ПОДТВЕРЖДЕНИЯ**

### Шаг 5: EDA (skill_eda.md)
Используй скилл eda.
Спроси фокус анализа или используй общий.

### Шаг 6: Итог
Покажи финальное summary:
```
📊 Готово!
- Kaggle: {file1}
- API: {file2}
- Merged: {file3}
- EDA: notebooks/{notebook}

Ключевые находки:
1. ...
2. ...
3. ...
```

## Критически важно
- **НЕ ПРОПУСКАЙ паузы** — каждый чекпоинт требует "да" от пользователя
- Если пользователь хочет изменить план — адаптируйся
- Если шаг упал — исправь и повтори, не пропускай
- Пользователь может сказать "пропусти kaggle" — тогда пропусти

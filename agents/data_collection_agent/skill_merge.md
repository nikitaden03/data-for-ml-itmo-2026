# Скилл: Merge Datasets

## Когда использовать
Есть 2+ скачанных датасета, нужно объединить в один.

## Как работать

### Шаг 1: Анализ
Попроси пользователя скинуть вывод этого кода (или вставь в свой скрипт):

```python
import pandas as pd
from pathlib import Path

data_dir = Path("./data")
for f in sorted(data_dir.glob("*.csv")):
    df = pd.read_csv(f, nrows=5)
    print(f"\n{'='*60}")
    print(f"📁 {f.name}")
    print(f"   Shape: {df.shape}")
    print(f"   Columns: {list(df.columns)}")
    print(f"   Dtypes:\n{df.dtypes.to_string()}")
    print(f"   Head:\n{df.head(3).to_string()}")
```

### Шаг 2: Определение стратегии

Проанализируй и выбери стратегию:

```
Вариант A: MERGE по ключу (SQL JOIN)
├── Когда: есть общий столбец (id, дата, страна...)
├── Типы: inner / left / outer
└── Пример: merge по country_code

Вариант B: CONCAT (склейка)
├── Когда: одинаковые столбцы, разные строки
├── Типы: вертикальный (append) / горизонтальный
└── Пример: данные за разные годы

Вариант C: СЛОЖНЫЙ MERGE
├── Когда: нет прямого ключа
├── Нужно создать ключ (нормализация названий, маппинг)
└── Пример: "United States" ↔ "US" ↔ "USA"
```

Объясни пользователю:
- Какую стратегию выбрал и почему
- По каким ключам объединяешь
- Что будет с пропусками
- Какой ожидается размер результата

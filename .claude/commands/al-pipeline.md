# Пайплайн: Active Learning Agent

Полный цикл эксперимента: подготовка данных → AL циклы для разных
стратегий → сравнение → отчёт.

## Вход

Пользователь даёт:
- Путь к датасету (CSV)
- Целевая переменная
- (опционально) Начальный размер N (дефолт: 50)
- (опционально) Количество итераций (дефолт: 5)
- (опционально) Размер батча на итерацию (дефолт: 10)
- (опционально) Какие стратегии сравнивать

## Инициализация

Создай структуру:
```
mkdir -p agents data/raw data/reports notebooks
```

## Выполнение

### Фаза 0: ПОДГОТОВКА ДАННЫХ

1. Загрузи датасет
2. Базовый анализ: shape, классы, баланс
3. Раздели данные:
   - **test set** — 20% стратифицированный, фиксированный
   - **initial labeled** — N=50 стратифицированный из оставшихся
   - **unlabeled pool** — всё остальное

Покажи:
```
📊 Подготовка данных:
   Total: {n} samples, {k} classes
   Test set: {n_test} samples
   Initial labeled: {n_init} samples ({per class})
   Unlabeled pool: {n_pool} samples
   
   Class distribution (initial labeled):
   {class_1}: {n} ({pct}%)
   {class_2}: {n} ({pct}%)

   Стратегии: entropy, random
   Итераций: 5, батч: 10 примеров
   
   Начинаем?
```

**⏸️ СТОП. ЖИЖДИ ПОДТВЕРЖДЕНИЯ.**

Пользователь может:
- Изменить N, количество итераций, батч
- Добавить/убрать стратегии
- Поменять модель

### Фаза 1: AL ЦИКЛ — ENTROPY

Запусти полный цикл:

```
Iteration 0: fit на initial (50) → evaluate → accuracy={}, F1={}
Iteration 1: query(entropy, 10) → fit (60) → evaluate → accuracy={}, F1={}
Iteration 2: query(entropy, 10) → fit (70) → evaluate → accuracy={}, F1={}
Iteration 3: query(entropy, 10) → fit (80) → evaluate → accuracy={}, F1={}
Iteration 4: query(entropy, 10) → fit (90) → evaluate → accuracy={}, F1={}
Iteration 5: query(entropy, 10) → fit (100) → evaluate → accuracy={}, F1={}
```

Показывай прогресс на каждой итерации.


## Критические правила

1. **Воспроизводимость** — random_state ВЕЗДЕ одинаковый (42)
2. **Честное сравнение** — один initial set, один test set для всех стратегий
3. **Test set не протекает** — тест не попадает в pool и labeled
4. **Pool уменьшается** — после query примеры ПЕРЕМЕЩАЮТСЯ из pool в labeled
5. **Паузы обязательны** — между фазами ждёшь подтверждения
6. **al_agent.py самодостаточный** — можно запустить отдельно от Claude Code
7. **Ноутбук выполнимый** — после создания запусти и проверь
8. **Стратифицированный split** — initial labeled должен содержать
   примеры ВСЕХ классов пропорционально
9. **Не перемешивай стратегии** — каждая стратегия работает
   со своей КОПИЕЙ pool-а, не мутируй общий
10. Все Юпитер ноутбуки должны быть с отработанными ячейками и выводами - ОБЯЗАТЕЛЬНО

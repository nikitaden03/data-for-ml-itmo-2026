# Полный ML Пайплайн

Оркестрирующий скилл. Запускает 4 пайплайна последовательно:
1. **Сбор и анализ** — collect-and-analyze-pipeline
2. **Качество данных** — data-quality-pipeline
3. **Авто-разметка** — labeling-pipeline
4. **Active Learning** — al-pipeline

Каждый пайплайн запускается только после завершения предыдущего.
Все паузы внутри пайплайнов сохраняются.

## Вход

Пользователь описывает задачу. Из его запроса определи:
- Тему/домен данных
- Источники (Kaggle / API / scraping)
- Целевую переменную
- Задачу ML (классификация / регрессия)

## Критические правила

1. **Строгая последовательность** — шаг N начинается только после завершения шага N-1
2. **Передача артефактов** — каждый шаг получает на вход артефакт предыдущего:
   - Шаг 2 берёт merged CSV из Шага 1
   - Шаг 3 берёт cleaned CSV (рекомендованная стратегия) из Шага 2
   - Шаг 4 берёт labeled CSV из Шага 3
3. **Все паузы сохраняются** — внутри каждого пайплайна паузы не пропускать
4. **Между шагами — тоже пауза** — переход к следующему шагу только после "да"
5. **Если шаг упал** — исправь и повтори, не пропускай шаг
6. **Пользователь может пропустить шаг** — если просит пропустить, предупреди что будет потеряно и что нужно предоставить вручную
7. **README для каждого шага** — финальный артефакт каждого пайплайна обязателен

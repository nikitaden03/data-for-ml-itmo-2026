# Пайплайн: Auto-Labeling Agent

Последовательно выполняет auto_label → generate_spec → check_quality → export.
Паузы для подтверждения на каждом этапе.

## Вход

Пользователь описывает:
- Данные (путь, модальность)
- Задачу разметки
- Конечную цель (обучение модели, аналитика, передача разметчикам)

## Инициализация

```
mkdir -p data/raw data/labeled data/export data/reports docs notebooks src
```

## Выполнение

### Фаза 1: AUTO LABEL

Выполни скилл auto-label.
- Определи модальность и задачу
- Предложи модель и классы
- **⏸️ ЖЖДИ подтверждения классов и модели**
- Размеь данные
- Покажи примеры и распределение

```
═══════════════════════════════════════
📋 ФАЗА 1: AUTO LABEL — ЗАВЕРШЕНА
═══════════════════════════════════════
Размечено: {n} записей
Классов: {n}
Средний confidence: {mean}
Uncertain (<0.5): {n} ({pct}%)

Перехожу к спецификации?
```

**⏸️ СТОП**

### Фаза 2: GENERATE SPEC

Выполни скилл generate-spec.
- Используй результаты авто-разметки для примеров
- High confidence → хорошие примеры
- Low confidence → граничные случаи
- **⏸️ ПОКАЖИ спецификацию и жижди правок**

```
═══════════════════════════════════════
📋 ФАЗА 2: ANNOTATION SPEC — ЗАВЕРШЕНА
═══════════════════════════════════════
Классов: {n}
Примеров на класс: {n}
Граничных случаев: {n}

Файл: docs/annotation_spec.md
Проверь и скажи если нужны правки.
```

**⏸️ СТОП**

### Фаза 3: CHECK QUALITY

Выполни скилл check-quality.
- Посчитай метрики
- Если можно — прогони вторую модель для agreement
- **⏸️ Если κ низкий — спроси нужно ли перезразметить**

```
═══════════════════════════════════════
📋 ФАЗА 3: QUALITY CHECK — ЗАВЕРШЕНА
═══════════════════════════════════════
Cohen's κ: {value} ({interpretation})
% Agreement: {pct}%
Label distribution: balanced/imbalanced
Uncertain records: {n}

Экспортировать в Label Studio?
```

**⏸️ СТОП**

Если качество плохое (κ < 0.4):
```
⚠️ Качество разметки низкое (κ = {value}).
Варианты:
1. Перезразметить другой моделью
2. Экспортировать для ручной разметки (все uncertain)
3. Сузить классы и попробовать снова
Что делаем?
```

**⏸️ СТОП**

### Фаза 4: EXPORT

Выполни скилл export-labelstudio.
- Конвертируй в JSON
- Сгенерируй XML конфиг
- Валидируй

```
═══════════════════════════════════════
📋 ФАЗА 4: EXPORT — ЗАВЕРШЕНА
═══════════════════════════════════════
Tasks: {n}
Format: Label Studio JSON
Validation: ✅
```

**⏸️ СТОП**

## Критические правила

1. **Паузы обязательны** — каждый чекпоинт ждёт пользователя
2. **Классы подтверждаются** — никогда не размечай без аппрува списка классов
3. **Спецификация ревьюится** — пользователь может уточнить определения
4. **Плохое качество = откат** — если κ < 0.4, предлагай перезапуск
5. **JSON валидируется** — не отдавай невалидный экспорт
6. **labeling_utils.py единый** — все скиллы дополняют, не дублируют
7. **Ноутбуки выполнимые** — после создания запусти и проверь
8. **README обязателен** — финальный артефакт каждого пайплайна; фиксирует решения, цифры и причины изменений классов
9. Все Юпитер ноутбуки должны быть с отработанными ячейками и выводами - ОБЯЗАТЕЛЬНО

# Active Learning Experiment — strategy_a

## Задача

Сравнить стратегии активного обучения на датасете отзывов о парфюмерии.
Проверить, позволяет ли умный выбор примеров для разметки экономить labeling effort.

## Данные

| Параметр | Значение |
|---|---|
| Датасет | `data/labeled/strategy_a_labeled.csv` |
| Объём | 807 отзывов на русском языке |
| Признаки | `review_text` → TF-IDF (5000 признаков, bigrams, sublinear_tf) |
| Таргет | `label_binary`: `liked` / `not_liked` |
| Баланс классов | liked 57.5% / not_liked 42.5% |

## Конфигурация эксперимента

| Параметр | Значение |
|---|---|
| random_state | 42 (везде) |
| Test set | 162 сэмплов (20%, стратифицированный) |
| Initial labeled | 50 сэмплов (стратифицированный) |
| Unlabeled pool | 595 сэмплов |
| Итераций | 5 |
| Батч | 10 сэмплов / итерация |
| Модель | Logistic Regression (C=1.0, max_iter=1000) |

## Стратегии

- **Entropy sampling** — на каждой итерации выбирает 10 примеров с наибольшей энтропией предсказаний модели (наиболее неуверенные)
- **Random baseline** — выбирает 10 случайных примеров из пула

## Результаты

| Итерация | N labeled | Entropy Acc | Entropy F1 | Random Acc | Random F1 | Delta F1 |
|---|---|---|---|---|---|---|
| 0 | 50 | 0.5741 | 0.3647 | 0.5741 | 0.3647 | 0.0000 |
| 1 | 60 | 0.5679 | 0.3750 | 0.5741 | 0.3647 | +0.0103 |
| 2 | 70 | 0.5802 | 0.4045 | 0.5741 | 0.3647 | +0.0398 |
| 3 | 80 | 0.6111 | **0.5982** | 0.5741 | 0.3647 | +0.2335 |
| 4 | 90 | 0.6296 | 0.5707 | 0.5741 | 0.3647 | +0.2060 |
| 5 | 100 | 0.6111 | 0.5516 | 0.5741 | 0.3647 | +0.1869 |

### Экономия на разметке

Entropy достигает плато Random (F1=0.3647) уже на **50 примерах** — экономия **50%** усилий на разметку.

При равном бюджете (100 примеров): Entropy F1=0.5516 vs Random F1=0.3647 (**+0.187**).

## Ключевые наблюдения

1. **Random baseline схлопывается в тривиальный предиктор.** При дисбалансе классов 57.5%/42.5% модель, обученная на случайных примерах, всегда предсказывает `liked`. Accuracy=0.574 — это просто частота мажоритарного класса. F1 остаётся на уровне 0.365 во всех итерациях.

2. **Entropy sampling ломает этот паттерн.** Выбирая примеры на границе решения, стратегия принудительно добавляет сложные случаи. Уже на итерации 3 (80 примеров) модель достигает F1=0.598 — в 1.64 раза выше, чем random.

3. **Нестабильность на итерациях 4-5** (F1 падает с 0.598 до 0.552) — нормальное поведение для малых датасетов. Entropy активно добавляет трудные/шумные примеры, что может временно ухудшать метрику.

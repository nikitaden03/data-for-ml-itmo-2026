# Active Learning Experiment — Russian Financial News

## Задача

Сравнение стратегий активного обучения (Entropy vs Random) на задаче бинарной классификации русскоязычных финансовых новостей (`positive_market_impact` 0/1).

---

## Данные

| Параметр | Значение |
|---|---|
| Датасет | `data/labeled/strategy_a_labeled.csv` |
| Строк | 13 799 |
| Признак | `body` — текст новости |
| Целевая переменная | `positive_market_impact` (0/1) |
| Баланс классов | 52.5% / 47.5% |
| Особенность | 73% строк без заголовка — использовался только `body` |

---

## Конфигурация эксперимента

| Параметр | Значение |
|---|---|
| `random_state` | 42 |
| Test set | 20% (2 760 строк, стратифицированный) |
| Initial labeled | 200 строк (стратифицированный) |
| Unlabeled pool | 10 839 строк |
| Batch size | 100 примеров за итерацию |
| Итераций | 10 |
| Модель | LogisticRegression (C=1.0, max_iter=1000) |
| Векторизатор | TF-IDF (max_features=10 000, ngram=(1,2), sublinear_tf=True) |

---

## Стратегии

**Entropy sampling** — на каждой итерации выбираются примеры с наибольшей энтропией предсказаний модели: -Σ p·log(p). Теоретически должен быстрее улучшать модель, концентрируясь на "пограничных" случаях.

**Random sampling** — случайная выборка из пула без учёта уверенности модели. Служит базовым бейзлайном.

## Ключевые наблюдения

1. **Random > Entropy на всём диапазоне итераций** — уже с итерации 1 Random превосходит Entropy и разрыв растёт. При 1 200 примерах разница составляет **-0.049 F1**.

2. **Причина: шум в метках** (~20% от GPT-4o silver labels). Entropy sampling выбирает наиболее неопределённые примеры, которые в данном датасете являются преимущественно *шумными*, а не информативными. Модель обучается на мусоре и деградирует.

3. **Общий низкий F1 (~0.50)** — TF-IDF на коротких Telegram-постах не захватывает достаточно сигнала. 73% строк без заголовка, средняя длина тела — 740 символов, но многие посты < 100 символов.

4. **Рекомендация**: для этого датасета Random sampling предпочтительнее Entropy. Чтобы Entropy работал лучше, необходимо: а) очистить метки (убрать шум), б) использовать более мощные эмбеддинги (ruBERT, sentence-transformers).
